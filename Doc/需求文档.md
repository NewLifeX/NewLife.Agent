# NewLife.Agent 需求文档

## 1. 系统定位

**NewLife.Agent** 是一个面向长期运行 .NET 应用的**跨平台服务守护与运维框架**，将控制台/Web/Worker 程序快速托管为操作系统服务，统一提供安装、启停、监控、重启与排障能力。

- **一句话描述**：继承 `ServiceBase`，你的程序就拥有了跨平台服务治理能力。
- **目标用户**：需要长期运行后台服务的 .NET 开发者与运维人员。
- **核心场景**：Windows 服务部署、Linux 守护进程、macOS LaunchAgent、ASP.NET Core 生产托管。

---

## 2. 愿景目标

### 2.1 核心愿景

成为 .NET 生态中**跨平台服务托管的首选框架**，填补 `Microsoft.Extensions.Hosting` 在服务安装与运维能力上的空白。

### 2.2 战略目标

| 目标 | 描述 | 衡量标准 |
|------|------|---------|
| **生态集成** | 与 ASP.NET Core、Worker Service 等主流框架零摩擦集成 | 提供 ≥2 个官方集成示例 |
| **运维增强** | 健康监控自愈能力持续领先 | 提供 Web 管理面板 |
| **可视化运维** | 通过 Web 面板管理多个服务实例 | 实现远程状态查询与启停控制 |

## 4. 竞品差异化定位

详见 [竞品分析报告](./竞品分析报告.md)，核心差异：

| 维度 | NewLife.Agent | Topshelf | WinSW | `Hosting` 系列 |
|------|:---:|:---:|:---:|:---:|
| 跨平台 | ✅ 全平台 | ❌ Windows only | ❌ Windows only | ⚠️ 需组合多包 |
| 内置运维命令 | ✅ 7 个命令 | ✅ 部分 | ✅ 全 | ❌ |
| 健康监控自愈 | ✅ 全内置 | ❌ | ❌ | ❌ |
| 框架兼容 | net40+ | net452+ | .NET Framework 4.6.1+ | net462+ |
| 维护状态 | 🟢 活跃 | 🔴 已归档 | 🟡 低活跃 | 🟢 官方 |

### 核心竞争壁垒

1. **唯一跨平台一体化方案**：同类竞品中唯一同时覆盖 Windows/Linux/macOS 的服务框架
2. **健康自愈不可替代**：竞品均不提供内存/线程/句柄监控与自动回收
3. **20 年框架兼容**：net40 ~ net10.0，覆盖所有 .NET 历史版本

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 框架本身开销 < 5MB 内存，服务启停延迟 < 3s |
| **可靠性** | WatchDog 检测间隔 ≤ 30s，假死检测准确率 > 99% |
| **兼容性** | 新增 API 需保持 net40 兼容（条件编译降级），不引入非必要外部依赖 |
| **安全性** | 服务安装/卸载不暴露敏感配置；命令行参数不含凭据 |
| **文档** | 每个公共 API 须有 XML 注释；重大变更更新运维手册 |
