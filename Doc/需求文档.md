# NewLife.Agent 需求文档

> 版本：v2.0 | 日期：2026-07-15

本文档描述 NewLife.Agent 的愿景、核心目标和功能方向。完成状态在[功能清单](功能清单.md)中追踪，详细设计在[架构设计](架构设计.md)中展开。

## 1. 背景与愿景

### 1.1 系统定位

**NewLife.Agent** 是一个面向长期运行 .NET 应用的**跨平台服务守护与运维框架**，将控制台/Web/Worker 程序快速托管为操作系统服务，统一提供安装、启停、监控、重启与排障能力。

- **一句话描述**：继承 `ServiceBase`，你的程序就拥有了跨平台服务治理能力。
- **目标用户**：需要长期运行后台服务的 .NET 开发者与运维人员。
- **核心场景**：Windows 服务部署、Linux 守护进程、macOS LaunchAgent、ASP.NET Core 生产托管。

### 1.2 愿景

成为 .NET 生态中**跨平台服务托管的首选框架**，填补 `Microsoft.Extensions.Hosting` 在服务安装与运维能力上的空白。让服务开发者只需关注业务逻辑，运维治理能力由框架统一提供。

## 2. 核心目标

| 编号 | 目标 | 层级 | 一句话描述 |
|------|------|------|-----------|
| M1 | 服务抽象与跨平台宿主 | 核心 | 提供统一的 `ServiceBase` 基类，自动适配 Windows/Linux/macOS 多平台服务管理 |

## 3. 功能需求

### 3.1 M1 服务抽象与跨平台宿主

- **M1-1 `ServiceBase` 基类**：继承即获得服务治理能力，统一 `ServiceName`/`DisplayName`/`Description` 元数据，管理运行状态 `Running`。
- **M1-2 Windows 服务**：使用 Win32 SCM API 实现完整的服务注册、启动、停止、卸载，支持电源/会话/时间变更系统事件。
- **M1-3 Windows 登录自启动**：通过注册表 Run 项实现用户登录后自动运行，适用于桌面场景。
- **M1-4 Linux Systemd**：自动生成 `.service` 单元文件，调用 `systemctl` 完成服务管理，支持丰富的 systemd 配置（重启策略、资源限制、网络依赖等）。
- **M1-5 Linux Procd（OpenWrt）**：面向路由器/嵌入式 Linux 的 procd init 脚本管理。
- **M1-6 Linux RcInit（传统 init.d）**：面向非 systemd 环境的 init.d 脚本管理。
- **M1-7 Linux SysVinit**：SysVinit 兼容识别。
- **M1-8 macOS LaunchAgent**：通过 `launchctl` 管理 `~/Library/LaunchAgents/*.plist`。
- **M1-9 平台自动检测与适配**：`Init()` 方法根据当前操作系统和配置自动选择最合适的宿主实现。
- **M1-10 兜底宿主**：无对应系统服务时以 `DefaultHost` 兜底运行。

### 3.2 M2 命令行运维体系

- **M2-1 命令工厂**：`CommandFactory` 通过反射自动发现 `BaseCommandHandler` 派生类，构建命令映射，处理命令冲突。
- **M2-2 安装服务（`-i`）**：将当前程序注册为系统服务。
- **M2-3 安装并启动（`-install`）**：安装后自动启动。
- **M2-4 重新安装（`-reinstall`）**：停止→卸载→安装→启动完整流程。
- **M2-5 卸载服务（`-u`）**：从系统中移除服务注册。
- **M2-6 停止并卸载（`-uninstall`）**：先停止服务再卸载。
- **M2-7 启动服务（`-start`）**：启动已安装的服务。
- **M2-8 停止服务（`-stop`）**：停止正在运行的服务。
- **M2-9 重启服务（`-restart`）**：重新启动服务。
- **M2-10 状态查询（`-status`）**：查看服务安装状态与运行状态。
- **M2-11 模拟运行（`-run`）**：前台控制台模式运行，用于调试。
- **M2-12 执行服务（`-s`）**：由系统服务管理器调用的真正服务执行入口。
- **M2-13 看门狗（`-watch`）**：立即执行一次看门狗检查，拉起被守护服务。
- **M2-14 交互菜单模式**：无参数启动时进入菜单循环，展示可执行命令。
- **M2-15 自定义命令扩展**：继承 `BaseCommandHandler` 定义 `Cmd`/`Description`/`ShortcutKey`，自动加入命令体系。

### 3.3 M3 健康监控与自愈

- **M3-1 内存阈值监控**：工作集内存超过 `MaxMemory`（MB）时自动重启进程释放资源。
- **M3-2 线程数阈值监控**：进程总线程数超过 `MaxThread` 时自动重启。
- **M3-3 句柄数阈值监控**：进程句柄数超过 `MaxHandle` 时自动重启（Windows）。
- **M3-4 定时自动重启**：达到 `AutoRestart` 设定分钟数且落入 `RestartTimeRange` 允许时间段时触发重启。
- **M3-5 WatchDog 看门狗**：按周期检查被守护服务列表，发现未运行则自动拉起。
- **M3-6 主动内存整理**：`FreeMemory()` 执行 GC 回收 + `EmptyWorkingSet` 释放虚拟内存。
- **M3-7 启动后命令**：服务启动后自动执行 `AfterStart` 配置的外部命令。

### 3.4 M4 配置管理

- **M4-1 `Setting` 配置模型**：基于 `Config<T>` 基类，`Agent.config` 文件持久化。
- **M4-2 服务元数据配置**：`ServiceName`/`DisplayName`/`Description` 独立配置。
- **M4-3 多实例部署**：同一程序通过不同 `ServiceName` 形成多个独立实例。
- **M4-4 配置文件热加载**：`Config<T>` 内置热加载机制，运行时修改配置即时生效。

### 3.5 M5 Web 管理面板

- **M5-1 内嵌 HTTP 服务器**：基于 `HttpServer` 的轻量 Web 服务器，默认端口 5580。
- **M5-2 状态查询 API**：返回服务运行状态、进程信息、系统资源、TCP 连接数、磁盘 IOPS 等。
- **M5-3 启停控制 API**：`start/stop/restart` 操作。
- **M5-4 配置读写 API**：读取所有配置项元数据（名称、类型、当前值），按白名单更新配置。
- **M5-5 密码修改 API**：校验旧密码后更新新密码，立即生效。
- **M5-6 日志读取 API**：按文件/行数/级别过滤读取日志内容，返回日志文件列表。
- **M5-7 Token 鉴权**：Bearer Token 签发与验证，24 小时过期。
- **M5-8 三级鉴权策略**：`None` 不鉴权 / `LocalOnly` 本地免鉴权 / `Full` 全部鉴权。
- **M5-9 登录爆破防护**：IP 级别失败记录与临时封锁。
- **M5-10 磁盘 IOPS 监控**：跨平台采集（Windows PDH/WMIC + Linux `/proc/diskstats`）。
- **M5-11 扩展面板注册**：通过继承 `AgentWebPanel` 重写 `GetExtensions()` 注册自定义面板。
- **M5-12 运行时凭据更新**：`UpdateCredentials()` 修改用户名密码，即时生效无需重启。
- **M5-13 前端界面**：内嵌 `index.html` 前端页面，支持主题切换。

### 3.6 M6 Host 集成扩展

- **M6-1 `ServiceLifetime`**：继承 `ServiceBase` 并实现 `IHostLifetime`，桥接 Agent 生命周期与 `Microsoft.Extensions.Hosting`。
- **M6-2 `UseAgentService()` 扩展方法**：HostBuilder 上的一键集成入口。
- **M6-3 `ServiceLifetimeOptions` 配置**：通过 Options 模式配置服务元数据。
- **M6-4 ASP.NET Core 集成**：将 ASP.NET Core 应用注册为系统服务，保留 Kestrel 管道和 DI 容器。
- **M6-5 Worker Service 集成**：将 `BackgroundService` 长驻任务纳入统一服务治理。

### 3.7 M7 框架兼容

- **M7-1 ~ M7-10**：覆盖 `net40;net45;net461;netstandard2.0;netstandard2.1;net6.0;net7.0;net8.0;net9.0;net10.0` 十种目标框架。
- **M7-11 条件编译降级**：低版本框架通过条件编译符号（`NETFRAMEWORK`/`NETSTANDARD2_0`/`NETCOREAPP` 等）降级 API 使用。
- **M7-12 项目模板**：`dotnet new service` 快速创建服务项目。

### 3.8 M8 平台工具类

- **M8-1 `ServiceHelper`**：运行时框架检测（dotnet/java）、从文件名分析工作目录。
- **M8-2 `ServiceModel` / `ServiceConfig`**：服务安装和查询的数据模型。
- **M8-3 `SystemdSetting`**：Systemd `.service` 单元文件生成，支持 Type/Environment/Restart/KillMode/OOMScoreAdjust 等配置。
- **M8-4 Windows 平台支持类**：`Advapi32`（SCM API P/Invoke）、`Desktop`（桌面进程启动）、`NativeMethods`（内存整理）、`PowerStatus`（电源状态）。

## 4. 🧊 暂缓清单

| 功能 | 暂缓理由 | 解冻前提 |
|------|----------|----------|
| 多实例统一管理视图 | Web 管理面板刚完成第一期，多实例管理需要更复杂的发现和通信机制 | Web 面板稳定运行后评估需求 |
| 健康指标历史趋势图 | 需要时序数据存储方案，当前 Web 面板仅展示实时数据 | 用户明确需求或性能优化需要 |
| Fluent API 配置风格 | Topshelf 风格的链式 API，当前继承式已满足需求 | 社区反馈强烈或主要用户提出 |

## 5. 不做什么

- 不做独立 exe 包装工具（如 WinSW/NSSM 模式）：NewLife.Agent 定位为库级别集成，非外部包装器
- 不做纯 XML 配置驱动：以代码配置为主，XML 文件仅作补充
- 不主动引入第三方 DI 容器：与 `Microsoft.Extensions.Hosting` 集成已满足生态需要

### 核心竞争壁垒

1. **唯一跨平台一体化方案**：同类竞品中唯一同时覆盖 Windows/Linux/macOS 的服务框架
2. **健康自愈不可替代**：竞品均不提供内存/线程/句柄监控与自动回收
3. **Web 管理面板**：唯一提供内置 Web 管理界面的服务框架
4. **20 年框架兼容**：net40 ~ net10.0，覆盖所有 .NET 历史版本
