# NewLife.Agent 竞品分析

> 版本：v1.0 | 日期：2026-07-15

## 1. 竞品格局

NewLife.Agent 定位为 **跨平台服务守护与运维框架**，将控制台/Web/Worker 程序快速托管为操作系统服务。同类方案可分为三类：

| 类型 | 代表 | 特点 |
|------|------|------|
| **一体化服务框架** | NewLife.Agent、Topshelf | 库级别集成，继承基类即可获得完整服务治理能力 |
| **外部包装工具** | WinSW、NSSM | 独立 exe，通过配置文件包装任意可执行程序为服务 |
| **平台内置抽象** | `Microsoft.Extensions.Hosting.*` | .NET 平台级抽象，需组合多个包才能覆盖多平台 |

## 3. 定位差异

| 维度 | NewLife.Agent | Topshelf | WinSW/NSSM | `Hosting` 系列 |
|------|:---:|:---:|:---:|:---:|
| 目标用户 | C#/.NET 服务开发者 | .NET Windows 开发者 | 运维人员（任意语言） | .NET 现代应用开发者 |
| 集成方式 | 继承 `ServiceBase` | Fluent API | 独立 exe + XML | DI 注册 + 扩展方法 |
| 运维能力 | 全内置 | 部分 | 全内置 | 需外部命令 |
| 监控自愈 | 全内置 | 无 | 无 | 无 |
| 维护状态 | 🟢 活跃 | 🔴 已归档 | 🟡 低活跃 | 🟢 官方 |

## 4. 核心优势

1. **唯一跨平台一体化方案**：单库覆盖 Windows/Linux/macOS 全部主流服务管理方式
2. **健康自愈不可替代**：竞品均不提供内存/线程/句柄监控与自动回收
3. **内置 Web 管理面板**：唯一提供内置 Web 管理界面的服务框架
4. **框架兼容性**：最低支持 net40，覆盖 20 年跨度

## 5. 差距与方向

| 维度 | 现状 | 可追赶方向 |
|------|------|-----------|
| NuGet 下载量 | 103.6K | Topshelf 因早期先发优势达 39.6M，需加大文档推广 |
| Fluent API | 不支持 | Topshelf 的链式 API 更符合现代 .NET 风格 |
| 第三方集成 | 较少 | 可提供更多与 MassTransit/Quartz.NET 等集成示例 |
